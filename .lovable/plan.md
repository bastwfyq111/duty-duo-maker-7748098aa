# خطة: معايير توزيع قابلة للتخصيص + تحسين ألوان الورديات

## 1. أوزان التوزيع القابلة للتخصيص

إضافة حقلين رقميين داخل نافذة التوزيع التلقائي يتيحان للمستخدم التحكم في أهمية كل معيار عند توزيع الورديات:

- **وزن ساعات العمل** — مدى أهمية موازنة إجمالي الساعات بين الموظفين.
- **وزن عدد الورديات** — مدى أهمية موازنة عدد الورديات (أيام العمل) بين الموظفين.

### التغييرات في `src/lib/auto-assign.ts`
- إضافة حقلين اختياريين إلى `AutoAssignConstraints`:
  - `weightHours?: number` (افتراضي 5)
  - `weightWorkUnits?: number` (افتراضي 2)
- تعديل دالة `candidateScore` لتستقبل القيود وتستخدم هذه الأوزان بدل الأرقام الثابتة الحالية (`hours + workUnits * 2`)، مع مضاعف يجعل وزن عدد الورديات مؤثراً مقابل الساعات (لأن الساعات أرقام أكبر بكثير من عدد الورديات).
- تمرير `c` إلى كل مواضع استدعاء `candidateScore` داخل حلقة التوزيع.

### التغييرات في `src/components/AutoAssignDialog.tsx`
- إضافة حالتين `weightHours` و`weightWorkUnits` مع قيم افتراضية.
- عرض حقلي إدخال رقمي (أو منزلقين) بعنوان واضح ضمن قسم العدالة في النافذة.
- تمرير القيمتين ضمن كائن `constraints` عند التوليد.

## 2. تعديل ألوان الورديات المضافة حديثاً

استبدال الألوان الـ12 الأخيرة (السطور 32–43 في `src/components/AddShiftDialog.tsx`، أسفل تعليق "ألوان جديدة") بألوان أوضح وأكثر تمايزاً بصرياً عن الألوان الموجودة، مع الحفاظ على صيغة HSL (`H S% L%`) لتوافق نظام الألوان.

## تفاصيل تقنية

```text
candidateScore الجديد:
  score  = hoursPerEmp[i]  * weightHours
  score += workUnits[i]    * (weightWorkUnits * 5)
  score += consecutive[i]  * 3
  score += (lastShift === code ? 12 : 0)
```

القيم الافتراضية تُبقي السلوك مقارباً للحالي حتى لا يتغيّر التوزيع لمن لا يعدّل الإعدادات.
